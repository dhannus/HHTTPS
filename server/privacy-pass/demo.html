Anonym Mensch.
Pseudonym mit Rolle.
Beides kombiniert.

Diese Demo zeigt das HHTTPS-Modell in Aktion: Privacy Pass als anonyme „ich bin ein Mensch"-Schicht (RFC 9578), ergänzt durch eine HHTTPS Role-Attestation als pseudonyme „ich habe Rolle X"-Schicht. Beide Layer werden vom Server kryptografisch unabhängig verifiziert und nur zusammen Zugriff auf den geschützten Endpunkt geben.

Privacy Pass Layer

Anonymer Token, VOPRF-P384, unlinkable.

Server sieht: „Ein Mensch hat einen gültigen Token vorgelegt." Mehr nicht.

HHTTPS Role Layer

Pseudonymer JWT, ES256, an WebAuthn gebunden.

Server sieht: „Dieser bekannte Authenticator hat Rolle X attestiert."
1

Anonymen Mensch-Beweis holen

Dein Browser baut lokal einen geblindeten Token, schickt ihn an unseren Privacy-Pass-Issuer, finalisiert das Ergebnis und hält damit einen anonymen Nachweis in der Hand. Der Issuer kann den Token später nicht mit deiner Identität verknüpfen.

Bereit. Klicke den Button.
Privacy Pass Token (146 Bytes)
2

Rolle attestieren (HHTTPS)

Du authentifizierst dich mit einem Passkey (WebAuthn) und wählst eine Rolle. Der HHTTPS-Issuer stellt dir einen ES256-JWT mit deiner pseudonymen Subject-ID und der Rolle aus.

Welche Rolle willst du attestieren?

Warte auf Schritt 1.
HHTTPS Role JWT (decoded)
3

Geschützten Endpunkt aufrufen

Du schickst beide Tokens an POST /privacy-pass/demo/protected. Der Endpunkt verifiziert Privacy Pass (kryptografisch) und HHTTPS Role (JWKS) unabhängig voneinander und gewährt nur Zugriff, wenn beide gültig sind.

Warte auf Schritt 2.
